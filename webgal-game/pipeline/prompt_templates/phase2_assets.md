# Phase 2: Asset Planner — Runtime Prompt Template

Orchestrator fills `{{PLACEHOLDERS}}` then sends as Agent prompt.

---

TASK: Generate asset manifest for all visual assets needed by this WebGAL game.

## Inputs (read from disk)
- `{{STATE_DIR}}/characters.json` — every character needs figure + mini avatar
- `{{STATE_DIR}}/scene_graph.json` — every scene needs a background
- Naming rules: bg_ prefix for backgrounds, figure_ prefix for sprites, cg_ prefix for event CGs

## Output: assets_manifest.json (project root)

```json
{
  "base_dir": "{{PROJECT_DIR}}/public/game",
  "model": "doubao-seedream-4-0-250828",
  "images": [
    {
      "filename": "bg_xxx",
      "subdir": "background",
      "size": "2560x1440",
      "prompt": "anime style, soft watercolor, <scene description>, no text, no watermark"
    },
    {
      "filename": "figure_xxx",
      "subdir": "figure",
      "size": "1440x2560",
      "prompt": "anime style, soft watercolor, full-body portrait of <name>, <appearance>, standing pose, clean plain white background, no text, no watermark"
    }
  ]
}
```

## Rules
- Backgrounds: 2560x1440, `bg_` prefix
- Character sprites: 1440x2560, `figure_` prefix, `clean plain white background` in prompt (for bg removal)
- CGs: 2560x1440, `cg_` prefix
- Mini avatars: auto-generated by make_avatar.py later — figure_tao_yuanming → miniavatar_tao_yuanming
- All prompts must include: `anime style, soft watercolor` + `no text, no watermark`
- Filenames WITHOUT .webp extension (script appends it)

## Boundaries
- Write manifest ONLY, do not run Python scripts (orchestrator handles that)
- Do NOT write scene files
- Do NOT modify planning artifacts

## Output
Write assets_manifest.json. Report: total asset count broken down by type.
